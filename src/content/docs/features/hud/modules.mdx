The HUD is comprised of modules each with their own configurations and states.

# Modules

## Coordinates
The coordinates module displays the player's current coordinates in the world. 
It can be configured to show either the XYZ coordinates or just the XZ coordinates.

### Settings
#### <code>xyz-display</code>
Toggles between xyz and xz coordinate display.   
<p><b>Options</b>: on, off</p>

## Distance
The distance module displays the distance from the player to the set destination.

## Destination
The destination module displays the coordinates for the set destination.

## Direction
The direction module displays the player's current facing direction.

## Tracking
The tracking module displays an arrow that points to the tracked target. It can be configured to prioritize the main target or track any target.

### Settings
#### <code>hybrid</code>
Enables tracking of any target, prioritizing the main target.   
<p><b>Options</b>: on, off</p>

#### <code>target</code>
Selects the target for the tracker to point towards.   
<p><b>Options</b>: player, dest</p>

#### <code>display-type</code>
Changes the visual style of the tracker.   
<p><b>Options</b>: simple, compact</p>

#### <code>show-elevation</code>
Enables showing the elevation difference to the target.   
<p><b>Options</b>: on, off</p>

## Time
The time module displays the current world time.

### Settings
#### <code>24hr-clock</code>
Toggles between a 24 and 12 hour clock.   
<p><b>Options</b>: on, off</p>

## Weather
The weather module displays the current weather status of the world.

## Speed
The speed module displays the player's current movement speed.

### Settings
#### <code>2d-calculation</code>
Sets the speed calculation type. When enabled, the Y level is excluded from the calculation.   
<p><b>Options</b>: on, off</p>

#### <code>display-pattern</code>
Sets the formatting string for the speed value.   
<p><b>Options</b>: 0.00</p>   
<p>See <a href="https://docs.oracle.com/javase/8/docs/api/java/text/DecimalFormat.html">DecimalFormat</a> for formatting options.</p>

## Angle
The angle module displays the player's current viewing angle.

### Settings
    #### <code>display</code>
    Sets the angle display type.   
    <p><b>Options</b>: yaw, pitch, both</p>


# Commands
import Command from '@/components/command/command.astro';
import Example from '@/components/command/example.astro';

Each command shown starts with <code>/hud modules</code>.

<Command></Command>

## <code>(empty)</code>
Displays the HUD module edit GUI.
<p>The menu displays a random preview of the module,
    toggle settings, HUD order settings, custom HUD options,
    and buttons to switch the module being edited.</p>
<Example command="/hud module" />

## <code>order &lt;module&gt; &lt;order&gt;</code>
Sets the display order for the selected module.  
<Example command="/hud module order time 2"/>
<p>Enter a number greater than one for the order.</p>

## <code>setting &lt;module&gt; &lt;module_setting&gt; &lt;value&gt;</code>
Sets a specific HUD module setting to the given value.
<Example command="/hud module setting speed 2d-calculation off"/>
<p>For a list of modules and their settings, <a href="#module" summary="Module and their settings.">Click Here.</a></p>

## <code>toggle &lt;module&gt;</code>
Toggles the enabled status of the selected module.
<Example command="/hud module toggle time"/>

## <code>reset (module)</code>
Restores all HUD modules to their default values unless a specific module is specified.  
<Example command="/hud module reset"/>
